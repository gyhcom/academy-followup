# 작업 티켓

이 문서는 Academy Follow-up을 단계별로 구현하기 위한 기능 단위 티켓 목록입니다.

원칙:

- 커밋은 가능한 한 티켓 단위로 나눕니다.
- 각 티켓은 독립적으로 리뷰 가능한 크기로 유지합니다.
- 파일럿 학원은 더배움프라임영수학원을 기준으로 합니다.
- MVP에서는 셀프서브 결제보다 파일럿 사용 가능성이 우선입니다.

현재 우선순위:

1. 사용자 측면 파일럿 안정화: `T-501`, `T-604`, `T-310`, `T-213`, `T-214`
2. 관리자 측면 운영 준비: `T-701`, `T-705`, `T-703`, `T-704`, `T-708`
3. 상용화 이후 확장: `T-702`, `T-706`, `T-707`, `T-709`, `T-710`
4. 세부 로드맵: [사용자 측면 / 관리자 측면 다음 티켓 로드맵](./next-ticket-roadmap.md)

## 원장 피드백 반영 기능 정리

2026-05-07 대화에서 확인한 실제 요구는 아래입니다.

- 주간 카드만으로는 부족하고, 하나의 달력에서 날짜를 누르면 그 날짜의 학생 스케줄이 보여야 합니다.
- 선생님이 날짜와 시간을 직접 선택해 보강 문자를 만들고, 문자 후 확인 버튼으로 보강 스케줄을 자동 등록해야 합니다.
- 수업 시작 시간에 바로 결석 처리하면 안 됩니다. 16:00 수업 학생이 16:10에는 안 왔지만 16:15에 올 수 있으므로 `확인 필요`, `지각`, `결석` 같은 예외 상태가 필요합니다.
- 현재 결석 공유는 선생님이 카톡방에 올리고 데스크가 확인해 연락하는 방식이라 불편합니다. 반별 출석부에서 결석이 바로 연락 흐름으로 이어져야 합니다.
- 스케줄 입력은 반 전체 일괄 등록과 학생 개인 일정 추가가 모두 필요합니다.
- 문자 발송 대상은 학부모와 학생을 선택할 수 있어야 합니다.
- 학생 전화번호를 식별자로 쓰고 싶은 요구가 있지만, 다른 학원 재원 여부가 노출될 수 있으므로 파일럿 MVP가 아니라 개인정보 검토가 필요한 후속 기능입니다.
