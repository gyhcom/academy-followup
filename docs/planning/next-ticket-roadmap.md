# 사용자 측면 / 관리자 측면 다음 티켓 로드맵

마지막 업데이트: 2026-06-11

이 문서는 현재 MVP 이후 다음 개발을 `학원 사용자`와 `플랫폼 관리자` 관점으로 나눠 보기 위한 실행 순서입니다. 상세 완료 기준은 `tickets.md`를 기준으로 봅니다.

## 사용자 측면

원장/선생님이 실제 학원에서 1~2주 쓰며 막히는 부분을 줄이는 티켓입니다. 모바일 390px 사용성을 1순위로 봅니다.

| 우선순위 | 티켓 | 상태 | 목적 | 완료 기준 |
| --- | --- | --- | --- | --- |
| 1 | T-501 실제 구조 데이터 운영 전환 검증 | 부분 완료 | 200명 seed 이후 실제 구조 20~30명으로 운영 흐름 검증 | CSV 투입 후 학생/반/스케줄/권한 정합성 확인 |
| 2 | T-604 운영 전환 릴리즈 체크리스트 | 완료 | 운영 시작 전 배포/권한/실데이터/SMS/Spring 상태 점검 | `docs/testing/operations-release-readiness-2026-06.md` 기준으로 확인 |
| 3 | T-310 SOLAPI 테스트 번호 제한 발송 | 예정 | dry-run 이후 테스트 번호 1건 실제 수신 검증 | 테스트 번호 수신, 로그, 실패 문구 확인 |
| 4 | T-213 데이터 내보내기 | 완료 | 학생/출석/문자/감사 로그 CSV 다운로드 | 관리 > 리포트 CSV 다운로드 가능 |
| 5 | T-214 학원 내부 감사 로그 | 부분 완료 | 주요 운영 변경 이력 추적 | 최근 변경 이력은 완료, 상세 검색/diff는 후속 |

## 관리자 측면

슈퍼어드민이 여러 학원을 운영, 지원, 상용화할 준비를 위한 티켓입니다. 학생 개인정보 상세 열람보다 학원 운영 상태 요약을 우선합니다.

| 우선순위 | 티켓 | 상태 | 목적 | 완료 기준 |
| --- | --- | --- | --- | --- |
| 1 | T-701 플랫폼 콘솔 운영 요약 고도화 | 예정 | 학원별 실제 사용 여부를 한눈에 판단 | 최근 로그인/출석/문자/월간 사용량 표시 |
| 2 | T-705 사용량 집계 | 예정 | 과금과 운영 모니터링에 쓸 월별 사용량 계산 | 학원별 이번 달 문자/학생/구성원 수 조회 |
| 3 | T-703 플랜/사용 제한 모델 | 예정 | 플랜별 제한과 안내를 시스템에 연결 | 한도 초과 시 경고 또는 제한 안내 |
| 4 | T-704 결제/구독 연동 | 예정 | 파일럿 이후 유료 전환 준비 | 테스트 결제로 구독 상태 반영 |
| 5 | T-708 보안/개인정보 운영 체크리스트 | 예정 | 개인정보 운영 기준 문서화 | 파일럿 학원에 설명 가능한 보안 기준 확보 |

## 추천 실행 순서

1. T-501 실제 구조 데이터 20~30명 전환 검증
2. T-310 SOLAPI 테스트 번호 제한 발송
3. T-708 보안/개인정보 운영 체크리스트
4. T-214 감사 로그 상세 검색/diff
5. T-701 플랫폼 콘솔 운영 요약 고도화
6. T-705 사용량 집계
7. T-703 플랜/사용 제한 모델
8. T-704 결제/구독 연동

## 운영 원칙

- 당장 목표는 상용 결제보다 친구 학원 파일럿 성공입니다.
- 실제 사용자는 모바일 브라우저가 1순위이고 PC는 보조입니다.
- SOLAPI 실제 발송은 테스트 번호 제한 후 점진적으로 확대합니다.
- 과금/구독은 파일럿 데이터와 실제 사용량이 확인된 뒤 진행합니다.
