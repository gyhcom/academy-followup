# 진행 체크

마지막 업데이트: 2026-05-12

## 현재 제품 방향

Academy Follow-up은 단순 문자 발송 도구가 아니라, 학생별 주간 스케줄과 팔로업 연락을 함께 관리하는 학원 운영 SaaS로 잡습니다.

현재 핵심 문장은 아래입니다.

> 반별 출석부에서 결석/지각을 확인하고, 학생별 스케줄을 보며 필요한 보강·학부모 연락을 바로 만드는 학원 운영 보드

최근 원장 피드백으로 `출석부 -> 결석/지각 문자 -> 보강 날짜 선택 -> 스케줄 자동 등록` 흐름을 파일럿 최우선 기능으로 올렸습니다.

## 현재 운영 상태

- GitHub `main`에 지금까지 만든 기능 브랜치를 모두 반영했습니다.
- 로컬/원격 브랜치는 `main`만 남도록 정리했습니다.
- 2026-05-12 기준 `npm run lint`, `npx tsc --noEmit`, `npm run build`를 통과했습니다.
- Vercel은 `main` push 기준으로 배포됩니다.
- 실제 Supabase DB 필수 컬럼은 파일럿 테스트 과정에서 적용 확인을 마쳤습니다.
- Production 모바일 UAT 결과는 `docs/testing/pilot-rehearsal-result-2026-05-11.md`에 기록했습니다.
- 2026-05-12 파일럿 안정화 점검에서 홈/문자/출석/관리/문자 템플릿 화면을 모바일 390px, 태블릿 768px, PC 1440px 기준으로 확인했습니다.
- 모바일 fixed 하단 탭바가 일부 환경에서 layout viewport 기준으로 밀리는 문제를 `visualViewport` 기준 높이로 보정했습니다.

## 친구에게 보여줄 현재 데모 흐름

파일럿 전 첫 시연은 아래 순서로 고정합니다.

```text
1. 랜딩 페이지에서 서비스 방향과 가격 확인
2. 원장 계정으로 로그인
3. 홈에서 운영 기준 날짜와 오늘 요약 확인
4. 출석부 탭에서 오늘 날짜와 수업 시간 선택
5. 학생을 확인 필요 또는 결석/지각으로 변경
6. 결석/지각 문자 초안 확인
7. 수신자를 학부모/학생/둘 다 중 선택
8. 문구 수정 후 dry-run 발송
9. 학생별 최근 팔로업 히스토리 확인
10. 보강 날짜와 시간을 선택해 보강 안내 생성
11. 보강 발송 후 스케줄 자동 등록 확인
12. 관리 탭에서 학생/반/구성원/문자 템플릿/학원 설정 확인
```

시연 중에는 실제 SMS 발송을 켜지 않습니다. `sms_dry_run=true` 상태에서 기록 저장과 중복 방지까지만 보여줍니다.

## 파일럿 전 필수 점검

1. Supabase 실제 DB에 최신 migration 컬럼이 모두 있는지 확인
2. 더미 학생/반/스케줄/출석 기록이 데모에 충분한지 확인
3. 원장 계정과 선생님 계정 각각 로그인 확인
4. 모바일 390px 폭에서 출석부와 관리 폼이 가로로 밀리지 않는지 확인
5. dry-run 발송 후 `followups`, `message_logs`에 기록이 남는지 확인
6. `SMS_DRY_RUN` 또는 학원 설정 `sms_dry_run`이 실제 발송을 막고 있는지 확인

## 부분 완료 또는 보류

- 문자 발송 API는 dry-run까지 완료, 실제 SMS 발송은 SOLAPI 환경변수와 `SMS_DRY_RUN=false` 전환 필요
- 반 비활성화는 `classes.status` 컬럼이 필요해 보류
- 선생님/구성원 등록과 테스트 계정 검증은 완료했지만, 이메일 초대 링크 방식은 보류
- 운영 보드의 선택 학생 주간 시간표는 완료
- 날짜 선택 후 보강 문자 생성과 문자 후 자동 스케줄 등록은 완료
- 출석부 DB 모델, 반별 출석부 화면, 결석/지각 문자 연결, 원장/데스크 출석 현황은 완료
- 전화번호 기반 학생 식별/학원 간 스케줄 공유는 개인정보 이슈가 있어 파일럿 MVP에서 제외
- 학생 CSV 일괄 등록은 기본 기능과 중복 미리보기까지 완료했지만, 실제 학원 데이터 30명 이상으로 추가 검증 필요
- 실제 운영용 감사 로그/이벤트 분석은 아직 미구현

## 현재 중요한 결정

- 출석부가 파일럿의 첫 사용 가치입니다. 결석을 카톡방에 공유하고 데스크가 따로 연락하는 흐름을 먼저 줄입니다.
- 수업 시작 후 미도착 학생을 바로 결석 확정하지 않습니다. `확인 필요` 상태를 두고, 4:10 미도착 후 4:15 도착 같은 예외를 수정할 수 있게 합니다.
