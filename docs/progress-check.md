# 진행 체크

마지막 업데이트: 2026-05-08

## 현재 제품 방향

Academy Follow-up은 단순 문자 발송 도구가 아니라, 학생별 주간 스케줄과 팔로업 연락을 함께 관리하는 학원 운영 SaaS로 잡습니다.

현재 핵심 문장은 아래입니다.

> 반별 출석부에서 결석/지각을 확인하고, 학생별 스케줄을 보며 필요한 보강·학부모 연락을 바로 만드는 학원 운영 보드

최근 원장 피드백으로 `출석부 -> 결석/지각 문자 -> 보강 날짜 선택 -> 스케줄 자동 등록` 흐름을 파일럿 최우선 기능으로 올렸습니다.

## 부분 완료 또는 보류

- 문자 발송 API는 dry-run까지 완료, 실제 SMS 발송은 SOLAPI 환경변수와 `SMS_DRY_RUN=false` 전환 필요
- 반 비활성화는 `classes.status` 컬럼이 필요해 보류
- 선생님/구성원 초대는 다음 관리 기능 이후 진행
- 운영 보드의 선택 학생 주간 시간표는 완료
- 날짜 선택 후 보강 문자 생성과 문자 후 자동 스케줄 등록은 완료
- 출석부 DB 모델, 반별 출석부 화면, 결석/지각 문자 연결, 원장/데스크 출석 현황은 완료
- 전화번호 기반 학생 식별/학원 간 스케줄 공유는 개인정보 이슈가 있어 파일럿 MVP에서 제외

## 현재 중요한 결정

- 출석부가 파일럿의 첫 사용 가치입니다. 결석을 카톡방에 공유하고 데스크가 따로 연락하는 흐름을 먼저 줄입니다.
- 수업 시작 후 미도착 학생을 바로 결석 확정하지 않습니다. `확인 필요` 상태를 두고, 4:10 미도착 후 4:15 도착 같은 예외를 수정할 수 있게 합니다.
- 완전한 캘린더 앱은 만들지 않지만, 보강은 실제 날짜 기준으로 선택해야 합니다.
- 외부 일정은 학생별 보강 불가 시간으로 취급합니다.
- 문자 발송은 바로 자동화보다 미리보기, 수정, 확인 후 발송을 우선합니다.
- 전화번호 기반 학생 식별/학원 간 스케줄 조회는 개인정보 리스크가 있어 상용화 이후 별도 검토합니다.
