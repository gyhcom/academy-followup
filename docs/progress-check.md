# 진행 체크

마지막 업데이트: 2026-05-07

## 현재 제품 방향

Academy Follow-up은 단순 문자 발송 도구가 아니라, 학생별 주간 스케줄과 팔로업 연락을 함께 관리하는 학원 운영 SaaS로 잡습니다.

현재 핵심 문장은 아래입니다.

> 학생별 수업·외부일정·보강 가능 시간을 한눈에 보고, 필요한 학부모 연락을 바로 만드는 학원 운영 보드

## 완료된 기능

### 기반

- Next.js 프로젝트 초기화
- GitHub repo 연결
- Vercel 배포 연결
- Supabase 프로젝트 연결
- Supabase SSR client/proxy 구성
- 서버 전용 service role client 구성
- `.env.local` Git 제외
- 기본 문서 작성

### 인증/워크스페이스

- Supabase 이메일/비밀번호 로그인
- 로그아웃
- 비로그인 `/app` 접근 차단
- 테스트 계정과 더배움프라임영수학원 profile 연결
- 학원명/카테고리 기반 워크스페이스 표시

### 운영 보드

- 반 목록 조회
- 학생 목록 조회
- 학부모 연락처 마스킹
- 학생별 빠른 사유 선택
- 문자 미리보기 API
- 문자 본문 수정
- 원문 복구
- 팔로업 기록 저장 API
- 저장 중/성공/실패 UI
- 모바일 하단 문자 작성 패널

### 랜딩/디자인

- PC/모바일 랜딩 분리
- 모바일 전용 헤더/히어로
- PC 전용 제품 목업
- 벤치마킹 기반 기능 맵 문서화
- 한국 경쟁 서비스 가격/구조 조사 문서화

### 관리 기능

- `/app` 내부 `운영 보드` / `관리` 탭 분리
- 원장/관리자만 관리 탭 접근
- 관리 홈에서 반/학생/구성원 현황 표시
- 반 등록/수정 API
- 반 등록/수정 폼
- 학생 등록/수정 API
- 학생 등록/수정 폼
- 학생 재원/휴원/퇴원 상태 관리
- 학부모 연락처 저장 전 정규화와 목록 마스킹
- 저장 후 `/app` 데이터 새로고침

## 부분 완료 또는 보류

- 문자 발송 API는 dry-run 구조만 초안 상태이며, 저장된 `followupId` 기준 발송 흐름은 아직 미연결
- 반 비활성화는 `classes.status` 컬럼이 필요해 보류
- 학생 등록/수정은 API와 폼까지 완료했고, 등록 후 주간 스케줄 입력 연결은 `T-216`에서 진행
- 선생님/구성원 초대는 다음 관리 기능 이후 진행
- 학생 스케줄은 제품 방향에 반영됐고, DB/화면 구현은 다음 단계

## 다음 개발 순서

1. `T-215` 학생 스케줄 DB 모델
2. `T-216` 학생별 주간 스케줄 등록/수정
3. `T-217` 학생 스케줄 보드
4. `T-218` 보강 문자 생성
5. `T-308` 문자 발송 dry-run
6. `T-208` 반 비활성화 스키마 결정
7. `T-206` 선생님/구성원 초대

## 현재 중요한 결정

- 완전한 캘린더 앱을 만들지 않습니다.
- 먼저 학생별 주간 반복 스케줄만 만듭니다.
- 외부 일정은 학생별 보강 불가 시간으로 취급합니다.
- 날짜별 예외, 드래그 앤 드롭, 학부모 캘린더 공유는 파일럿 검증 후 판단합니다.
