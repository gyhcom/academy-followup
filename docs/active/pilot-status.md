# 파일럿 상태

## 1. 현재 파일럿 준비 상태

구현된 기능:

- 원장/관리자/선생님/보조 선생님 로그인과 권한 분리
- 학생/반/구성원/스케줄 관리
- 반별 출석부와 도착/지각/결석/확인 필요/보강 상태
- 개별 문자와 전체문자 dry-run
- 중복 수신자 제외
- 문자 길이와 SMS/LMS 예상 표시
- 학생 히스토리
- 타 학원 스케줄 공유와 보강 제외 표시
- 운영 리포트/CSV 내보내기
- 감사 로그와 최근 변경 이력

확인된 기능:

- 200명 파일럿 seed 정합성 정상화
- assistant 발송 제한 UAT
- 전체문자 대상/중복 제외 카운트 미리보기
- 모바일 390px 기준 주요 화면 UAT

아직 확인 필요한 기능:

- 실제 원장/선생님이 1주일 쓰는 동안의 클릭 수와 문구 피로도
- 실제 학원 데이터 전환 시 CSV/스케줄/권한 정합성
- SOLAPI 테스트 번호 제한 실제 발송
- 담당 수업이 없는 teacher/assistant 빈 상태 안내

고객 피드백:

- 현재는 친구 원장 피드백을 받으며 모바일 홈/출석/문자 흐름을 조정 중입니다.
- 큰 기능 추가보다 파일럿 중 막히는 UX와 문구 수정이 우선입니다.

## 2. 데이터 상태

200명 seed 기준 기대값:

- 학생 200명
- 반 20개
- 정규 스케줄 580개
- 스케줄 미등록 0명

현재 기준:

- Supabase SQL Editor에서 200명 seed 정리/verify 완료
- 파일럿 기준 데이터는 200명 seed로 고정
- 자세한 확인 SQL과 결과는 [파일럿 seed 정합성 문서](../testing/pilot-seed-consistency-2026-06.md)를 기준으로 봅니다.

## 3. UX 상태

- 모바일 390px이 1순위입니다.
- PC/iPad는 가로 overflow와 주요 버튼 접근 가능 여부만 보조 확인합니다.
- 주요 개선 후보는 [모바일 UX 점검 보고서](../testing/mobile-ux-polish-review-2026-06.md)에 기록되어 있습니다.
- 고객 피드백 전까지 큰 레이아웃 개편은 보류합니다.

## 4. 파일럿 전 확인 체크리스트

- 로그인: owner/teacher/assistant 계정 접근
- 홈: 오늘 일정, 미처리, 공유 일정, 날짜 선택
- 출석: 수업 선택, 도착, 지각, 결석, 확인 필요, 보강
- 문자: 개별 문자, 전체문자, 중복 제외, byte 표시, 테스트 발송
- 보강: 우리 학원/타 학원/공유 일정 충돌 차단
- 관리: 학생, 반, 구성원, 스케줄, 템플릿, 설정
- 리포트: 요약과 CSV 다운로드
- 이력: 감사 로그 표시
- 권한별 화면: 관리 탭 숨김, 전화번호 비노출, assistant 발송 제한
- 모바일 390px: 가로 overflow, 하단 탭, 바텀시트, 긴 리스트
