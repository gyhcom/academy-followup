# T-434 플랫폼 학원 생성 안정화 결과

## 확인 범위

- `/platform` 새 학원 생성 폼
- `/api/platform/academies` 생성 전 중복 검증
- 생성 직후 원장 전달 정보 표시
- 학원 목록 초기 연결 상태 표시
- 슈퍼어드민 권한 등록 SQL 문서화

## 구현 결과

- 학원명 입력 시 slug 후보가 자동 입력됩니다.
- 원장 임시 비밀번호를 버튼으로 자동 생성할 수 있습니다.
- 학원 생성 성공 후 로그인 주소, 원장 이메일, 임시 비밀번호, slug가 전달 카드로 표시됩니다.
- 학원 목록에는 `원장 연결됨`, `설정 생성됨`, `반 0개`, `학생 0명`, `/app 연결 준비` 상태가 표시됩니다.
- 생성 전 이미 사용 중인 slug와 이미 Auth에 존재하는 원장 이메일을 차단합니다.
- 새 학원은 `academy_settings.sms_dry_run=true` 기본값으로 시작합니다.

## 수동 확인 절차

1. 슈퍼어드민 계정으로 `/platform`에 접속합니다.
2. `학원 생성`을 열고 학원명, 원장 이름, 원장 이메일을 입력합니다.
3. `임시 비밀번호 자동 생성`을 눌러 비밀번호를 채웁니다.
4. 학원을 생성하고 전달 카드의 로그인 정보를 확인합니다.
5. 생성된 원장 이메일/임시 비밀번호로 `/login`에서 로그인합니다.
6. `/app`에서 생성한 학원 워크스페이스가 열리는지 확인합니다.

## 주의 사항

- 임시 비밀번호는 생성 직후 전달 카드에서만 확인하는 운영을 전제로 합니다.
- 기존 Auth 이메일을 새 학원 원장으로 재사용하는 정책은 아직 만들지 않았습니다.
- 슈퍼어드민 계정 자체는 Supabase SQL Editor에서 `platform_admins`에 수동 등록합니다.
