## 4. 모바일 핵심 흐름

### 홈

- 오늘 학원 일정이 먼저 보임
- 공유된 타 학원 일정이 `공유됨`, `보강 제외`로 구분됨
- 공유 일정에는 상대 학원명/전화번호가 노출되지 않음
- 일정이 많으면 `전체 보기` 바텀시트에서 검색/필터 가능
- 상단 `이전` 버튼으로 직전 탭 복귀 가능

### 출석

- 수업 선택 가능
- 학생별 `도착` 토글 ON/OFF 가능
- `지각` 빠른 체크 가능
- `결석`, `확인 필요`, `보강` 예외 처리 가능
- 결석/지각 선택 후 문자 초안으로 이어짐

### 문자

- 개별 연락에서 반/학생 선택 가능
- 학생 선택 후 모바일 하단 CTA 또는 바텀시트로 초안 확인 가능
- byte, SMS/LMS 예상 표시 확인
- 2000byte 초과 시 저장/발송 차단
- 전체문자에서 중복 수신자 제외 ON/OFF 확인
- dry-run 저장 후 학생 히스토리에 남는지 확인

### 관리

- 원장 시작 순서가 보임
- 선생님 등록, 반 만들기, 학생 등록, 수업 시간 등록 접근 가능
- 반 학년에서 `무학년제`, `중2`, `중3` 선택 가능
- 학생 상세에서 타 학원 스케줄 공유 동의 체크 가능
- 타 학원 수업 직접등록과 연결 학원 공유 일정 확인 가능

## 5. 학원 간 스케줄 공유

권장 테스트 방식:

- 더배움 원장은 일반 브라우저
- 다솜 원장은 시크릿 창 또는 다른 브라우저 프로필

확인:

1. 양쪽 학생의 이름/학교/학년/전화번호가 같고, 양쪽 모두 공유 동의 ON이면 자동 연결
2. 한쪽 동의 OFF면 연결되지 않음
3. 전화번호가 다르면 연결되지 않음
4. 연결 후 홈에는 `공유된 타 학원 일정`으로 표시
5. 연결 후에도 상대 학원명/전화번호는 숨김
6. 한쪽에서 공유 해제하면 양쪽 모두 공유 일정이 사라짐
7. 보강 시간 선택 시 연결 학원 일정과 겹치면 차단

## 6. 시연 중 중단 기준

아래 문제가 나오면 실제 파일럿 시작 전에 수정합니다.

- 모바일 390px에서 가로 스크롤 발생
- 출석 저장이 눈에 띄게 지연되거나 실패
- 문자 dry-run 저장 실패
- 중복 제외 카운트가 실제 수신자와 맞지 않음
- 선생님/보조 선생님에게 전화번호 숫자 노출
- 공유 일정에 상대 학원명/전화번호 노출
- `sms_dry_run=false` 상태로 실제 번호 발송 위험 발생

## 원장 전달용 요약

- 파일럿 전에는 실제 문자를 보내지 않고 dry-run으로 출석/문자/기록 흐름을 검증합니다.
- 모바일에서 홈, 출석, 문자, 관리 순서로 눌러보면 됩니다.
- 타 학원 공유 일정은 보강 시간을 잡지 말아야 하는 시간으로 표시되고, 상대 학원명과 전화번호는 공개되지 않습니다.
- 실제 문자 발송은 발신번호와 SOLAPI 설정을 확인한 뒤 테스트 번호 1개부터 진행합니다.
