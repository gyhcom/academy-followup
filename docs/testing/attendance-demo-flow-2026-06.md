# 출석부 중심 데모 흐름

작성일: 2026-06-20

## 목적

원장님 우선순위 확정 전까지 신규 기능을 추가하지 않고, 현재 구현된 출석부와 문자 처리 흐름을 안정적으로 보여주기 위한 데모 기준입니다.

## 데모 전 확인

- `sms_dry_run=true` 상태를 유지합니다.
- `allow_assistant_send=false` 상태를 유지합니다.
- Vercel Production에는 `NEXT_PUBLIC_BACKEND_API_URL`을 설정하지 않습니다.
- 실제 학부모 번호로 SMS를 발송하지 않습니다.
- PC 1440px 화면을 우선으로 확인하고, 모바일 390px은 깨짐 여부만 확인합니다.

## 시연 순서

1. 원장 계정으로 로그인합니다.
2. `출석부` 메뉴로 진입합니다.
3. `달력 보기`에서 날짜를 선택합니다.
4. 선택 날짜 패널에서 수업 수, 학생 수, 출석, 지각, 결석, 미체크 요약을 확인합니다.
5. 요약 숫자를 눌러 학생 목록이 해당 상태로 좁혀지는지 확인합니다.
6. `오늘 처리` 화면으로 이동합니다.
7. 수업을 선택하고 학생 출석 상태를 확인합니다.
8. 지각 또는 결석 학생을 여러 명 선택합니다.
9. 일괄 문자 패널에서 공통 본문과 `{{studentName}}` 이름 치환 안내를 확인합니다.
10. `기록 저장` 또는 `테스트 발송` 흐름을 확인합니다.
11. 학생 row를 눌러 drawer를 열고 선택 날짜 수업 정보, 주간 스케줄, 출석 이력, 최근 연락 기록을 확인합니다.

## 현재 보여줄 수 있는 것

- 날짜 선택 기반 출석부 달력
- 선택 날짜의 수업별/학생별/연락 필요 목록
- 오늘 처리 화면의 수업별 출석 장부
- 지각/결석 학생 다중 선택
- 학생 이름 변수 `{{studentName}}` 치환 안내
- 연락 기록 저장 및 테스트 발송 흐름
- 학생 drawer의 기본 정보, 수업 정보, 최근 연락 기록

## 이번 데모에서 하지 않는 것

- 교재비/수강료 관리
- 월말 비용 안내 문자
- 학생 기준 월간 matrix 출석부
- 학생 개인 메모장 신규 기능
- 한 반 전체 출석 처리 버튼
- 새로운 DB 테이블 또는 API 추가
- 실제 학부모 대상 SMS 발송

## 중단 기준

- 달력 cell 텍스트나 badge가 cell 밖으로 삐져나오는 경우
- PC 1440px에서 가로 스크롤이 생기는 경우
- 지각/결석 다중 선택 후 문자 패널이 열리지 않는 경우
- `sms_dry_run=false`로 실제 발송 상태가 켜져 있는 경우
- 학생/반/스케줄 데이터가 데모에 충분하지 않은 경우
