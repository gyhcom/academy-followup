# T-414 모바일 우선 출석 UX 점검 결과

날짜: 2026-05-18

## 목적

출석 탭을 PC 대시보드가 아니라 선생님이 핸드폰으로 수업 중 빠르게 누르는 화면으로 재정리했습니다.

이번 작업은 DB/API/출석 저장 로직을 변경하지 않고, 모바일 390px 기준 화면 밀도와 조작 흐름만 개선했습니다.

## 변경 요약

- 상단 제목, 설명, 날짜 컨트롤 높이를 줄였습니다.
- 날짜 기준 요약을 작은 4칸 스트립으로 유지했습니다.
- 수업 선택 행을 얇은 리스트로 압축하고 선택 상태를 좌측 라인으로 표시했습니다.
- 학생 행은 학생명, 학년/학교, 저장 상태, 도착 토글 중심으로 재배치했습니다.
- 모바일에서는 학부모 연락처를 기본 행에서 숨기고, PC 이상에서만 보이게 했습니다.
- 예외 버튼과 `지각 / 결석 / 확인 필요 / 보강` 액션 높이를 줄였습니다.
- 결석/지각 문자 패널은 출석 체크를 방해하지 않는 보조 영역처럼 문구와 패딩을 정리했습니다.

## 확인 기준

- 모바일 390px에서 날짜, 요약, 수업 리스트가 과하게 길지 않아야 합니다.
- 수업 선택 후 학생 6~8명 기준으로 이전보다 더 많은 학생 행이 한 화면에 보여야 합니다.
- 도착 토글 ON/OFF는 기존 `present/pending` 저장 로직을 그대로 사용해야 합니다.
- `예외` 펼침 후 지각, 결석, 확인 필요, 보강 저장 흐름이 유지되어야 합니다.
- 결석/지각 선택 시 문자 패널, 수신자 선택, byte 표시, dry-run 흐름이 유지되어야 합니다.
- iPad 세로와 PC 폭에서는 가로 overflow 없이 기능 접근이 가능해야 합니다.

## 실행 검사

- `npm run lint`
- `npx tsc --noEmit`
- `npm run build`
- `git diff --check`

## 남은 확인

실제 운영 데이터가 있는 Supabase 환경에서 원장/선생님 계정으로 모바일 390px 출석 흐름을 다시 눌러보는 확인이 필요합니다.

특히 수업당 학생 수가 10명 이상일 때, 예외 버튼을 펼친 상태에서 행 밀도가 충분한지 한 번 더 확인합니다.
