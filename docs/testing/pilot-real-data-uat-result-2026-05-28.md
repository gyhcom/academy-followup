# 실제 구조 데이터 모바일 UAT 결과 - 2026-05-28

## 범위

- 실제 개인정보 대신 실제 학원 운영 구조를 닮은 30명 더미 CSV를 준비했습니다.
- 기준 파일: `docs/testing/pilot-real-structure-sample-2026-05-28.csv`
- 모바일 390px 기준으로 현재 운영형 demo 데이터에서 홈, 출석, 문자, 관리, 공유 일정 흐름을 확인합니다.
- 실제 SMS 발송은 하지 않고 `sms_dry_run=true` 전제를 유지합니다.

## 실제 구조 더미 CSV

구성:

- 중2 수학 A반: 12명
- 중3 영어 B반: 18명
- 총 30명
- 타 학원 공유 동의 학생: 16명

운영 전 입력 순서:

1. 관리 탭에서 반을 먼저 생성합니다.
   - `중2 수학 A반`
   - `중3 영어 B반`
2. CSV 일괄 등록으로 `pilot-real-structure-sample-2026-05-28.csv`를 넣습니다.
3. 반 공통 스케줄을 등록합니다.
   - 중2 수학 A반: 월/수/금 16:30-19:00, 수학
   - 중3 영어 B반: 화/목 19:30-21:30, 영어
4. 필요하면 학생 상세에서 타 학원 수업 또는 공유 동의를 확인합니다.

## 발견된 보정 후보

1. 실제 CSV를 넣은 뒤 관리 학생 목록에서 30명 이상 검색/필터 체감 확인이 필요합니다.
2. 실제 수업 시간 기준으로 출석부 수업 선택 리스트가 너무 길어지는지 확인이 필요합니다.
3. 전체문자 중복 제외 카운트는 실제 같은 보호자 번호가 있는 데이터로 추가 확인이 필요합니다.
4. SOLAPI 실제 발송은 아직 보류하고, 테스트 번호 1개 제한 발송을 별도 T-310에서 진행합니다.
5. 선생님/보조 선생님 계정으로 같은 데이터가 담당 반 기준으로 보이는지 확인이 필요합니다.

## 원장에게 전달할 요약

- 실제 학원 구조에 맞춘 30명짜리 CSV 샘플을 준비했습니다.
- 이 CSV는 실제 개인정보 없이 반, 학교, 학년, 연락처, 타 학원 공유 동의 흐름을 테스트하기 위한 파일입니다.
- 모바일 기준으로 홈, 출석, 문자, 관리, 공유 일정 흐름은 현재 demo 데이터에서 막히지 않았습니다.
- 다음 확인은 친구 학원 실제 반명/시간표에 맞춰 CSV를 넣고, 1개 수업에서 출석 체크와 문자 dry-run을 실제로 눌러보는 것입니다.
