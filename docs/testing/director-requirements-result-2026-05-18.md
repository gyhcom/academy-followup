# 원장 추가 요구사항 반영 결과 - 2026-05-18

## 범위

원장 대화에서 나온 요구사항을 `출석`, `전화번호 비노출`, `학생/반/스케줄 등록`, `학원 간 스케줄 공유`, `초기 세팅`으로 나눠 티켓화했습니다.

이번 구현은 파일럿 사용성에 직접 영향이 큰 아래 두 가지를 먼저 반영했습니다.

- T-417 출석 행에 지각 빠른 체크 추가
- T-418 선생님/보조 선생님 전화번호 비노출

## 반영 내용

| 항목 | 결과 | 메모 |
| --- | --- | --- |
| 도착 옆 지각 빠른 체크 | 완료 | `예외`를 열지 않아도 지각 처리 가능 |
| 지각 문자 연결 | 유지 | 기존 `late` 상태와 문자 초안 흐름 사용 |
| 결석/확인 필요/보강 | 유지 | 기존처럼 `예외` 안에서 처리 |
| 선생님 전화번호 비노출 | 완료 | 마스킹 숫자 대신 연락처 등록 상태만 표시 |
| 보조 선생님 전화번호 비노출 | 완료 | 선생님과 동일 기준 |
| 원장/관리자 전화번호 확인 | 유지 | 관리 화면에서는 기존 전화번호 표시 유지 |
| 학원 간 스케줄 공유 | 티켓화 | 개인정보 검토가 필요해 MVP 구현 제외 |

## 확인 기준

- 모바일 390px 출석 행에서 `지각`과 `도착`이 바로 보여야 합니다.
- `지각` 클릭 시 학생 상태가 `late`로 저장되고, 결석/지각 문자 패널 연결이 유지되어야 합니다.
- 선생님/보조 선생님 계정에서는 학생/학부모 전화번호 숫자가 보이면 안 됩니다.
- 원장/관리자 계정의 관리 화면에서는 운영 입력을 위해 전화번호 확인이 가능해야 합니다.
- 문자 발송 API는 기존처럼 서버에서 학생 ID 기준으로 전화번호를 조회해야 합니다.
