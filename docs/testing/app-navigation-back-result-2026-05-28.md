# 앱 상단 뒤로가기 결과 - 2026-05-28

## 목적

하단 탭만으로는 홈, 문자, 출석, 관리 화면을 오가며 이전 화면으로 돌아가기 불편한 문제를 보정했습니다.

## 반영 내용

- `/app` 내부 화면 이동 히스토리 stack을 추가했습니다.
- 이전 화면이 있을 때만 상단 `이전` 버튼을 표시합니다.
- 버튼에는 `홈/문자/출석/관리로 돌아가기` 보조 문구를 표시합니다.
- 같은 화면을 다시 누르면 히스토리를 중복으로 쌓지 않습니다.
- 관리 권한이 없는 계정에서 관리 화면으로 돌아가야 하는 경우는 홈으로 보정합니다.
- 뒤로가기 후 기존 탭 전환처럼 화면 상단으로 스크롤합니다.

## 확인 항목

- 홈 → 출석 이동 후 `이전` 클릭 시 홈으로 돌아갑니다.
- 홈 → 출석 → 문자 이동 후 `이전` 클릭 시 출석, 다시 클릭 시 홈으로 돌아갑니다.
- 오늘 일정의 출석 아이콘으로 이동한 출석 화면에서도 `이전`으로 홈에 돌아갑니다.
- 모바일 390px에서 뒤로가기 바는 하단 탭과 겹치지 않습니다.

## 제한

- 브라우저 URL 히스토리는 이번 범위에서 연동하지 않았습니다.
- 학생 상세, 문자 작성 같은 내부 바텀시트는 기존 닫기 버튼을 사용합니다.
