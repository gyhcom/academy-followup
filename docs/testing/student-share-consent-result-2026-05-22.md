# T-428 학생 등록 시 타 학원 스케줄 공유 동의 UX 보강 결과

## 결과

- 학생 등록/수정 폼에 `타 학원 스케줄 공유 동의 확인` 체크 항목을 추가했습니다.
- 동의 확인 전에는 학생 상세에서 공유 코드 생성과 코드 입력 연결을 비활성화했습니다.
- 공유 코드 생성 API는 현재 학원 학생의 동의 확인 상태를 검사합니다.
- 공유 코드 연결 API는 양쪽 학생의 이름·학교·학년 일치와 동의 확인 상태를 모두 검사합니다.
- 연결된 학원은 실제 학원명 대신 `연결 학원 1`, `연결 학원 2`로 표시합니다.
- 공유 스케줄 조회는 동의 확인이 유지된 학생의 활성 일정만 반환합니다.

## 종이 동의서 보강 문구

항목명: `타 학원 스케줄 공유 동의`

목적: `보강·상담 일정 조율을 위한 중복 수업 시간 확인`

제공 항목: `학생 식별 확인용 이름·학교·학년, 수업 요일·시간·일정 유형`

제공받는 자: `보호자 동의 후 연결된 제휴 학원`

보유 기간: `학원 간 연결 해제 또는 수강 종료 시까지`

안내 문구: `이 동의는 선택 사항이며, 동의를 거부해도 수강 자체에는 불이익이 없습니다.`

## 확인 기준

- 동의 미확인 학생: 공유 코드 생성/연결 불가
- 동의 확인 학생: 기존 공유 코드 생성/연결 가능
- 연결 전: 다른 학원 존재 여부와 개수 미노출
- 연결 후: 실제 학원명 대신 익명 라벨과 바쁜 시간대만 표시
- 공유 해제: 한쪽에서 해제하면 양쪽 모두 상대 스케줄 조회 불가

## 운영 메모

- 파일럿에서는 전자서명 없이 원장이 종이 동의서 수령 여부를 수동 체크합니다.
- 실제 보호자 전자 동의 링크나 인증 코드는 상용화 이후 후속 기능으로 분리합니다.
- 운영 DB에는 `supabase/migrations/20260522090000_add_student_schedule_share_consent.sql` 또는 같은 내용의 복구 SQL을 먼저 적용해야 합니다.
