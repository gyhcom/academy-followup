# 학원 간 학생 스케줄 공유 구현 결과

날짜: 2026-05-19

## 구현 범위

- 같은 SaaS 안의 학원끼리만 학생 스케줄 공유를 지원합니다.
- 전화번호 검색은 만들지 않습니다.
- 기본 공유 방식은 학생 저장 시 자동 연결이며, 공유 코드는 자동 연결이 되지 않는 예외 상황의 보조 기능입니다.
- 공유 코드는 연결하려는 두 학원 중 어느 쪽 원장/관리자든 발급할 수 있습니다.
- 보호자/학생 동의는 파일럿 단계에서 원장이 종이 동의서 수령 여부를 수동 확인한 것으로 기록합니다.
- 동의 확인 전에는 공유 코드 생성/연결을 막습니다.
- 연결 후에는 실제 상대 학원명 대신 `연결 학원 1` 같은 익명 라벨과 날짜/요일, 시간, 일정 제목, 일정 유형만 조회합니다.
- 보호자명, 전화번호, 반명, 선생님명, 메모는 공유 응답에 포함하지 않습니다.
- 공유 해제는 한쪽 학원이 해제하면 양쪽 모두 끊기는 정책입니다.

## 확인해야 할 흐름

1. A학원 원장이 학생 상세에서 공유 코드를 발급합니다.
2. B학원 원장이 자기 학원 학생 상세에서 해당 코드를 입력해 연결합니다.
3. 양쪽 학생 상세에서 익명 연결 학원 라벨과 상대 스케줄이 보입니다.
4. 보강 달력에서 연결 학원 일정과 시간이 겹치면 선택 불가 경고가 보입니다.
5. 원장/관리자는 연결 해제가 가능하고, 선생님/보조 선생님은 조회만 가능합니다.
6. 공유 코드의 학생과 현재 선택한 학생의 이름·학교·학년이 다르면 연결이 차단됩니다.
7. 어느 한쪽 학원에서 공유 해제하면 양쪽 학생 상세에서 연결 학원 스케줄이 사라집니다.

## 자동 연결 확인 흐름

1. A학원 학생과 B학원 학생의 이름, 학교, 학년, 학부모 연락처 또는 학생 연락처를 같게 입력합니다.
2. 양쪽 학생 모두 `타 학원 스케줄 공유 동의 확인`을 체크합니다.
3. 마지막으로 저장한 학생 기준으로 자동 공유 연결이 생성되는지 확인합니다.
4. 한쪽 학생의 전화번호를 다르게 수정하면 기존 연결이 해제되는지 확인합니다.
5. 한쪽 학생의 동의 체크를 끄면 기존 연결이 해제되는지 확인합니다.

## 타 학원 더미 데이터 테스트 절차

1. Supabase Authentication에서 `other-owner@test.com / 1234` 사용자를 생성합니다.
2. Supabase SQL Editor에서 `supabase/seed-shared-schedule-demo.sql`을 전체 실행합니다.
3. `owner@test.com / 1234`로 로그인해 더배움 학생 상세에서 공유 코드를 만듭니다.
4. 로그아웃 후 `other-owner@test.com / 1234`로 로그인합니다.
5. 앱 상단 학원명이 `다솜수학학원`인지 확인합니다.
6. `관리 > 학생 > 김민준` 상세에서 더배움 공유 코드를 입력합니다.
7. 연결 후 양쪽 학생 상세에서 `연결 학원 1` 라벨과 화/목 `19:30-20:30` 일정이 보이는지 확인합니다.
8. 더배움 문자 탭의 보강 달력에서 같은 시간대 선택이 차단되는지 확인합니다.
9. 반대로 다솜에서 공유 코드를 만들고 더배움에서 입력해도 연결되는지 확인합니다.
10. 한쪽에서 `공유 해제`를 누른 뒤 양쪽에서 연결이 사라지는지 확인합니다.

## 동일 학생 검증

- 정상 연결 기준: 더배움 `김민준 / 탕정중 / 중2` 공유 코드는 다솜 `김민준 / 탕정중 / 중2` 학생에게만 연결됩니다.
- 차단 기준: 다솜의 다른 학생 상세에서 같은 공유 코드를 입력하면 `공유 코드의 학생 정보와 현재 선택한 학생 정보가 다릅니다. 이름, 학교, 학년을 확인해 주세요.` 메시지가 표시되어야 합니다.
- 학교명이나 학년이 비어 있으면 같은 학생으로 자동 판단하지 않습니다. 학생 정보를 먼저 맞춘 뒤 다시 연결합니다.

## 동의 확인 기준

- 학생 등록/수정 화면에서 `타 학원 스케줄 공유 동의 확인`을 체크한 학생만 공유 코드 생성/연결이 가능합니다.
- 양쪽 학생 중 한쪽이라도 동의 확인이 꺼져 있으면 공유 코드 연결을 차단합니다.
- 공유 조회도 동의 확인이 유지된 학생의 활성 일정만 표시합니다.

## 세션 테스트 주의

- 같은 브라우저에서 `owner@test.com`과 `other-owner@test.com`을 동시에 로그인하면 Supabase 세션 쿠키가 공유되어 오래된 학생 상세 화면에서 학생을 못 찾을 수 있습니다.
- 양쪽 학원을 동시에 비교할 때는 일반 창과 시크릿 창, 또는 서로 다른 브라우저 프로필을 사용합니다.
- 세션이 바뀐 오래된 화면에서는 `현재 로그인한 학원과 선택한 학생 정보가 맞지 않습니다. 다른 학원 계정으로 로그인했거나 화면이 오래된 상태일 수 있습니다. 새로고침 후 다시 확인해 주세요.` 메시지가 보여야 합니다.

## 남은 후속

- 보호자 인증코드 또는 동의 링크 기반 실제 동의 플로우
- 여러 학원 연결이 늘어났을 때 연결 목록 검색/정렬
- 실제 다중 학원 데이터 seed는 `supabase/seed-shared-schedule-demo.sql`로 분리했습니다.
