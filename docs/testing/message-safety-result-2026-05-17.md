# 문자 안전 테스트 결과 - 2026-05-17

## 범위

- 실제 SMS 발송은 켜지 않고 `sms_dry_run=true` 기준으로 확인합니다.
- 점검 대상은 문자 초안, 출석 문자 패널, 템플릿 편집, `/api/followups`, `/api/messages/send`입니다.

## 반영 내용

- 문자 길이 계산 공통 기준을 추가했습니다.
  - 90byte 이하: SMS 예상
  - 91~2000byte: LMS 예상
  - 2000byte 초과: 저장/발송 차단
- 문자 작성 화면과 출석 문자 패널에 `자수 · byte · SMS/LMS 예상`을 표시합니다.
- 문자 템플릿 편집 화면에도 같은 길이 기준을 표시합니다.
- `/api/followups`는 2000byte 초과 본문 저장을 거부합니다.
- `/api/messages/send`는 발송 직전 2000byte 초과 본문을 다시 차단합니다.
- `/api/messages/preview`는 미리보기 응답에 길이 지표를 포함합니다.

## 수동 확인 시나리오

1. 원장 계정으로 로그인 후 문자 탭에서 학생과 사유를 선택합니다.
2. 짧은 문자를 입력하면 `SMS 예상`으로 표시되는지 확인합니다.
3. 한글 본문을 길게 늘려 90byte를 넘기면 `LMS로 발송될 수 있습니다`가 보이는지 확인합니다.
4. 2000byte를 넘기면 저장 버튼이 비활성화되는지 확인합니다.
5. 출석 탭에서 결석/지각 학생 문자 패널도 같은 표시와 차단이 적용되는지 확인합니다.
6. 관리 탭의 문자 템플릿 편집에서 2000byte 초과 시 저장 버튼이 비활성화되는지 확인합니다.
7. 같은 학생/같은 사유/같은 수신자에게 이미 발송된 기록이 있으면 `/api/messages/send`가 409로 차단하는지 확인합니다.
8. `sms_dry_run=true` 상태에서는 실제 SOLAPI 호출 없이 `message_logs`에 `dry_run` 로그가 남는지 확인합니다.

## 남은 실제 발송 전 확인

- SOLAPI 환경변수와 발신번호가 준비된 뒤 `T-310 SOLAPI 실제 발송`에서 테스트 번호로 실제 수신을 확인합니다.
- 실제 발송 테스트 전에는 운영 학원의 `sms_dry_run` 값을 임의로 끄지 않습니다.
