# 운영 세팅 리허설 결과 - 2026-05-13

## 목적

원장이 실제 운영 시작 전에 `구성원 등록 -> 반 생성 -> 학생 등록/반 배정 -> 수업 스케줄 입력 -> 출석부 확인` 흐름을 따라갈 수 있는지 확인했습니다.

이번 리허설은 UI 흐름과 반응형 안정성 확인이 목적이며, 실제 Supabase 운영 DB에 데이터를 자동 생성하지 않았습니다.

## 점검 환경

- 대상 URL: `http://localhost:3000`
- 계정: `owner@test.com / 1234`
- 검증 도구: Playwright
- 화면 폭:
  - 모바일 390px
  - iPad 768px
  - PC 1440px

## 확인 결과

| 항목 | 결과 | 메모 |

| 원장 로그인 | 통과 | `/login`에서 `/app` 진입 확인 |
| 관리 탭 진입 | 통과 | 하단/상단 탭에서 관리 화면 진입 확인 |
| 초기 세팅 화면 | 통과 | `운영 초기 세팅` 표시 |

| 학생 관리 액션 | 통과 | 반 배정/스케줄 미등록 안내 노출 |

| 콘솔 오류 | 통과 | 관련 console error 없음 |

## 발견 사항

- 기존 기능 조각은 이미 존재합니다.
  - 구성원 등록/수정
  - 반 생성/수정과 주 담당 선생님 배정
  - 학생 등록/수정과 소속 반 배정
  - 학생별 스케줄 추가/수정/삭제
  - 반 단위 수업 스케줄 일괄 등록
- 다음 검증에는 실제처럼 충분한 수업/출석 데이터가 필요합니다.
- 따라서 운영 리허설용 `supabase/seed-demo-operations.sql`을 별도 seed로 분리했습니다.

## 다음 액션

1. Supabase SQL Editor에서 `supabase/seed-demo-operations.sql` 실행
2. 원장 계정으로 홈/문자/출석/관리 화면 재확인
3. 오늘 기준 미체크/출석/지각/결석/확인 필요/보강 상태가 자연스럽게 보이는지 확인
4. 친구 원장 3분 데모 시나리오에 맞춰 문구와 화면 이동 보정
