# T-416 모바일 390px 파일럿 UAT 결과

날짜: 2026-05-18

## 목적

모바일 390px 기준으로 `홈 -> 출석 -> 문자 -> 관리` 핵심 파일럿 흐름을 실제로 눌러보고, 파일럿 데모와 1주일 테스트에 직접 영향을 주는 UI 문제만 좁게 보정했습니다.

## 확인 환경

- 계정: `owner@test.com`
- URL: 로컬 Next.js production start
- Viewport: 모바일 390px
- 실제 SMS 발송: 미실행, dry-run 기준

## 확인 결과

| 항목 | 결과 | 메모 |
| --- | --- | --- |
| 홈 첫 화면 | 통과 | 오늘 할 일, 운영 기준 날짜, 요약 숫자 표시 |
| 출석 탭 | 통과 | 수업 리스트, 학생 도착 토글, 예외 버튼 표시 |
| 출석 도착 토글 | 통과 | ON/OFF 조작 가능 |
| 출석 예외 펼침 | 통과 | 지각/결석/확인 필요/보강 버튼 표시 |
| 문자 탭 | 보정 | 탭 전환 시 이전 스크롤 위치가 유지되어 중간 위치에서 시작하는 문제 확인 |
| 관리 탭 | 통과 | 초기 세팅, 학생/반/스케줄 CTA 접근 가능 |
| 가로 overflow | 통과 | 390px에서 `scrollWidth === clientWidth` 확인 |
| 콘솔 에러 | 통과 | 관련 앱 콘솔 에러 없음 |

## 보정 내용

- 탭 전환 또는 홈 바로가기 이동 시 화면을 상단으로 복구하도록 수정했습니다.
- 이전 탭에서 아래로 스크롤한 뒤 문자 탭으로 이동해도 화면 중간이 아니라 문자 탭 상단부터 보입니다.
