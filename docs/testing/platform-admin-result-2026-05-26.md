# T-433 슈퍼어드민과 학원 원장 권한 분리 결과

## 구현 범위

- `platform_admins` 기반 슈퍼어드민 검증 helper 추가
- `/platform` 플랫폼 관리자 콘솔 추가
- `/api/platform/academies` 학원 생성/상태 변경 API 추가
- 로그인 후 이동 정책 변경
  - 학원 프로필이 있으면 `/app`
  - 플랫폼 관리자만 있으면 `/platform`
  - 둘 다 있으면 `/app` 우선, 앱 헤더에 `플랫폼 관리` 링크 표시

## 확인 기준

- 슈퍼어드민 계정은 `/platform`에 접근 가능해야 한다.
- 일반 원장 계정은 `/platform` 접근 시 차단되어야 한다.
- 슈퍼어드민이 학원 생성 폼으로 학원과 원장 계정을 함께 만들 수 있어야 한다.
- 생성된 원장 계정으로 로그인하면 `/app`에서 해당 학원 워크스페이스만 보여야 한다.
- 플랫폼 콘솔은 학생 개인정보 상세가 아니라 학원명, 상태, 플랜, 원장, 반/학생/구성원 수 요약만 보여야 한다.

## 운영 메모

- 슈퍼어드민 권한 부여는 Supabase SQL Editor에서 `platform_admins`에 user id를 추가해 처리한다.
- v1은 학원 생성/상태 관리까지이며 과금, 감사 로그, 플랫폼 전체 상세 분석은 후속으로 둔다.
