# 실제 구조 데이터 모바일 UAT 준비 결과 - 2026-05-22

## 목적

친구 학원 실제 데이터 20~30명 기준으로 `홈 -> 출석 -> 문자 -> 관리 -> 공유/보강` 흐름을 확인하기 전, 화면 안내와 CSV 입력 구조를 정리했습니다.

## 반영 내용

- 학생 상세의 타 학원 스케줄 공유 안내를 자동 연결 기준 중심으로 보강했습니다.
- 연결된 학원 카드에 `동의 확인`, `이름·학교·학년·전화번호 기준`, `상대 학원명 비공개` 배지를 표시합니다.
- CSV 일괄 등록 템플릿에 `타학원공유동의` 컬럼을 추가했습니다.
- CSV 값이 `동의`, `Y`, `true`, `1`, `확인`이면 공유 동의 확인으로 저장합니다.
- CSV 일괄 등록으로 들어온 동의 학생도 저장 후 자동 공유 매칭이 실행됩니다.

## 실제 구조 CSV 기준

필수 컬럼:

```csv
학생명,반,학교,학년,학부모명,학부모연락처,학생연락처,타학원공유동의,상태
```

입력 기준:

- `반`은 시스템에 이미 등록된 반 이름과 정확히 같아야 합니다.
- `타학원공유동의`는 동의 확인 학생만 `동의`로 입력합니다.
- 실제 개인정보가 부담되면 이름과 전화번호만 테스트용 값으로 바꾸고, 반/학교/학년/요일 구조는 실제와 비슷하게 둡니다.

## 모바일 UAT 확인 순서

1. 관리 탭에서 반 1~2개를 먼저 생성합니다.
2. CSV 일괄 등록으로 학생 20~30명을 넣습니다.
3. 반 단위 요일 묶음 스케줄을 등록합니다.
4. 출석 탭에서 수업별 학생 목록, 도착, 지각, 결석, 확인 필요, 보강을 확인합니다.
5. 문자 탭에서 개별 문자, 전체문자, 중복 제외, byte 표시, dry-run 저장을 확인합니다.
6. 같은 학생을 다른 학원 테스트 계정에도 등록해 자동 공유가 연결되는지 확인합니다.
7. 보강 달력에서 우리 학원 일정과 연결 학원 일정이 겹칠 때 선택이 차단되는지 확인합니다.
