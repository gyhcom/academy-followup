# 파일럿 UAT 체크리스트

이 문서는 친구 학원 파일럿 전 최종 확인용 체크리스트입니다.

목표는 데모가 아니라 실제 수업 후 운영 흐름이 끊기지 않는지 확인하는 것입니다.

## 1. 로그인/세션

- `/login` 접속 가능
- 테스트 계정 로그인 가능
- 비로그인 `/app` 접근 차단
- 로그아웃 후 `/app` 재접근 차단
- 모바일 Safari/Chrome 로그인 가능
- 세션 만료 후 오류 메시지 확인 가능

## 2. 학원 워크스페이스

- 로그인 후 학원명 표시
- 원장/선생님 권한 표시
- 반 목록 정상 표시
- 학생 목록 정상 표시
- 긴 반 이름이 UI를 깨지 않음

## 3. 출석부

- 날짜 선택 가능
- 반 선택 가능
- 수업 시간 선택 가능
- 학생 목록 시간순 표시
- 상태 변경 가능
- 상태 저장 후 새로고침 유지
- `확인 필요` 상태 사용 가능
- `지각` 상태 변경 가능
- `결석` 상태 변경 가능
- `보강` 상태 사용 가능
- 잘못 누른 상태 수정 가능

## 4. 문자 초안

- 학생 선택 시 문자 초안 생성
- 결석/지각별 문구 변경
- 학원명 치환 확인
- 학생명 치환 확인
- 선생님명 치환 확인
- 본문 수정 가능
- 너무 긴 문구가 화면을 깨지 않음

## 5. 수신자 선택

- 학부모 선택 가능
- 학생 선택 가능
- 학부모+학생 선택 가능
- 학생 번호 없을 때 학생 발송 비활성화
- 발송 전 수신자 확인 가능

## 6. dry-run 발송

- dry-run 저장 가능
- followup 기록 생성
- message log 저장
- 중복 발송 경고 표시
- 발송 실패 상태 확인 가능
- 실제 문자 비용 발생 없음

## 7. 학생 히스토리

- 최근 팔로업 표시
- 결석/지각 기록 표시
- 보강 기록 표시
- 최신순 정렬
- 학생 전환 시 히스토리 갱신

## 8. 보강 스케줄

- 달력 표시
- 날짜 선택 가능
- 시작 시간 입력 가능
- 종료 시간 입력 가능
- 보강 문자 생성 가능
- 보강 스케줄 자동 등록 가능
- 중복 보강 등록 방지
- 학생 상세에서 보강 확인 가능

## 9. 원장 운영 확인

- 오늘 출석 현황 표시
- 확인 필요 학생 표시
- 지각 학생 표시
- 결석 학생 표시
- 연락 미처리 학생 표시
- 선생님별 상태 확인 가능

## 10. 권한 분리

### 원장 계정

- 전체 반 조회 가능
- 전체 학생 조회 가능
- 관리 메뉴 접근 가능
- 설정 메뉴 접근 가능

### 선생님 계정

- 담당 반 중심 조회
- 제한된 관리 메뉴
- 설정 변경 불가

### 보조 선생님

- 발송 제한 동작 확인
- 초안 작성 가능 여부 확인

## 11. 모바일 사용성

- 390px 폭에서 가로 스크롤 없음
- 반 선택 칩 터치 가능
- 학생 선택 영역 충분한 높이 유지
- 버튼 터치 영역 충분
- 정보 카드와 액션 버튼이 시각적으로 구분됨
- 클릭 가능한 요소에는 동사 라벨 또는 액션 아이콘이 있음
- 하단 버튼이 홈 인디케이터와 겹치지 않음
- 문자 textarea 입력 가능
- 모바일 문자 작성 바텀시트가 열릴 때만 본문 스크롤 잠금
- 모바일 바텀시트 열린 상태에서 PC 폭으로 변경 시 스크롤 잠금 해제
- 달력 셀이 너무 작지 않음
- 출석 상태 버튼 겹치지 않음

## 12. 태블릿/PC 사용성

- iPad 세로/가로 정상 표시
- 운영 보드 컬럼 깨짐 없음
- 홈 빠른 이동은 카드 전체가 아니라 `열기` 버튼으로 동작함
- 출석 요약 패널은 클릭 가능한 카드처럼 보이지 않음
- PC 문자 화면에서 학생 선택 후 페이지 전체 스크롤 가능
- PC 문자 작성 패널 sticky 유지
- 긴 이름 줄바꿈 처리
- 원장 현황 카드 overflow 없음
- 출석 현황 패널 정렬 정상

## 13. 보안/개인정보

- 비로그인 API 401
- 다른 학원 데이터 접근 불가
- 전화번호 마스킹 정상
- 서버 로그에 민감정보 과다 노출 없음
- secret key 프론트 노출 없음

## 14. 파일럿 전 최종 확인

- Supabase 실제 DB 필수 컬럼 확인
- `followups.recipient_type`, `message_logs.recipient_type`, `students.student_phone` 존재 확인
- `profiles.status`, `profiles.phone`, `academy_settings` 존재 확인
- 선생님 2~3명 계정 준비
- 테스트 학생 30명 내외 입력
- 테스트 반 데이터 입력
- dry-run 활성화 확인
- 실제 발송 비활성화 확인
- 모바일 실사용 테스트 완료
- 원장 데모 흐름 리허설 완료

## 14-1. 초기 세팅/배정 프로세스

- 관리 탭에서 `초기 세팅` 섹션 확인
- 초기 세팅 순서가 `구성원 -> 반 -> 학생 -> 반 스케줄 -> 학생별 예외 일정`으로 보임
- 구성원 등록 버튼이 구성원 폼으로 연결
- 반 등록 버튼이 반 폼으로 연결
- 학생 등록 버튼이 학생 폼으로 연결
- 스케줄 미등록 보기 버튼이 학생 목록의 스케줄 미등록 필터로 연결
- 반 공통 스케줄 버튼이 반 관리로 연결
- 반 생성/수정 시 주 담당 선생님 배정 가능
- 학생 생성/수정 시 소속 반 배정 가능
- 학생 상세에서 개인 스케줄 추가 가능
- 학생 상세에서 기존 스케줄 수정 가능
- 학생 상세에서 기존 스케줄 삭제 가능
- 삭제된 스케줄이 운영 보드, 보강 후보, 출석 기준에서 제외됨
- 보조 선생님 다중 배정은 이번 파일럿 범위 밖이라는 안내가 문서에 남아 있음

## 15. 데모 시연 순서

추천 시연 흐름:

```text
랜딩 페이지 가격/가치 제안 확인
-> 원장 계정 로그인
-> 원장 오늘 미처리 보드 확인
-> 출석부 탭 이동
-> 오늘 날짜와 수업 시간 선택
-> 확인 필요/결석/지각 처리
-> 문자 초안 생성
-> 수신자 선택
-> 문구 수정
-> dry-run 발송
-> 학생 히스토리 확인
-> 보강 날짜/시간 선택
-> 보강 문자 생성
-> 보강 스케줄 자동 등록
-> 관리 탭에서 학생/반/구성원/설정 확인
```

시연 목표:

> 카톡/엑셀/개인 메모 없이 수업 후 후속 업무를 한 흐름으로 처리할 수 있다는 것을 보여준다.

## 16. 데모 중 말하지 않을 것

- 실제 SMS 발송 가능 여부를 과하게 강조하지 않음
- 전체 학원 200명 즉시 도입을 전제로 말하지 않음
- 수납, 성적, 문제은행 같은 미구현 기능을 약속하지 않음
- "AI 자동화"처럼 현재 제품 범위를 벗어나는 표현을 쓰지 않음

## 17. 데모 후 바로 물어볼 질문

- 수업 직후 선생님이 실제로 누를 수 있을 만큼 단순한가?
- 결석/지각 문자를 기존보다 빨리 만들 수 있는가?
- 원장 입장에서 오늘 미처리 보드가 카톡방 확인을 줄여주는가?
- 보강 날짜/시간 선택이 기존 방식보다 편한가?
- 파일럿을 한다면 먼저 넣을 반과 학생은 어느 범위가 적당한가?
