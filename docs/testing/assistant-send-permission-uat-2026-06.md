# T-620 Assistant 발송 제한 실계정 UAT 결과

## 1. 검수 목적

200명 운영형 pilot 데이터 기준으로 보조 선생님 계정의 실제 권한 흐름을 확인했습니다. 목적은 `assistant@test.com`이 담당 반 출석/팔로업 기록은 처리할 수 있지만, 학원 설정 `allow_assistant_send=false`에서는 문자 발송 또는 dry-run 발송이 서버에서 차단되는지 검증하는 것입니다.

## 2. 검수 환경

- 기준 브랜치: `codex/t-620-assistant-send-permission-uat`
- 기준 URL: `https://academy-followup.vercel.app/app`
- 기준 데이터: 더배움프라임영수학원 200명 pilot seed 적용 후 정리 완료 상태
- 기준 화면 폭: 모바일 390px
- 테스트 계정:
  - 원장: `owner@test.com / 1234`
  - 보조 선생님: `assistant@test.com / 12345678`
- 학원 설정:
  - `sms_dry_run=true`
  - `allow_assistant_send=false`
- 검수 방식:
  - 모바일 390px headless Chromium
  - Supabase service role read-only 확인
  - 발송 제한은 로그인 세션에서 `/api/messages/send` 직접 호출로 확인

## 4. Assistant 계정 검수 결과

### 홈

- 하단 탭은 `홈 / 문자 / 출석`만 표시되고 `관리` 탭은 보이지 않았습니다.
- 홈 첫 화면에서 보조 선생님 역할 문구가 표시되었습니다.
- 가로 overflow는 없었습니다. `scrollWidth=390`, `clientWidth=390`.

### 문자

- 문자 탭에서 담당 반 4개, 담당 학생 40명만 표시되었습니다.
- 학생/학부모 전화번호 숫자는 노출되지 않았고 `학부모 연락처 등록됨`처럼 상태 문구만 보였습니다.
- 학생 선택 후 결석 사유를 누르면 문자 초안 바텀시트가 열렸습니다.
- 초안에는 `90자 · 149byte · LMS로 발송될 수 있습니다.` 길이 정보가 표시되었습니다.
- 팔로업 기록 저장은 성공했습니다.

검수에 사용한 draft:

- 학생: `강서우136`
- 사유: `결석`
- followup id: `6c6da15b-a878-4bff-9e9e-1e7f8c1a609c`
- 저장 결과: `200`
- 저장 상태: `draft`
- 작성자: `assistant@test.com`

### 발송 제한

저장된 followup으로 `/api/messages/send`를 호출했습니다.

| 항목 | 결과 |
| --- | --- |
| 응답 코드 | `403` |
| 응답 메시지 | `이 팔로업 기록을 발송할 권한이 없습니다.` |
| followup 상태 | `draft` 유지 |
| followup sent_at | `null` |
| 해당 followup message_logs | `0건` |
| 전체 message_logs | 기존 `4건` 유지 |

따라서 `allow_assistant_send=false` 상태에서 assistant의 dry-run/실발송은 서버에서 차단됩니다.

### 출석

- 담당 반 출석 기록은 assistant 계정으로 수정 가능했습니다.
- 검수 payload는 `중3 수학 심화 A / 김서준081 / 2026-06-01 19:30-21:30 / present`입니다.
- 응답 코드: `200`
- 반환 record의 `teacherId`는 assistant 계정 id로 저장되었습니다.
- 담당 외 반 출석 수정 시도는 `403`으로 차단되었습니다.
- 담당 외 반 차단 메시지: `이 반의 출석부를 수정할 권한이 없습니다.`

### 리포트/관리 접근

- assistant 하단 탭에는 `관리`가 보이지 않았습니다.
- assistant 로그인 세션으로 `/api/reports/summary?range=today` 호출 시 `403`으로 차단되었습니다.
- 응답 메시지: `운영 리포트는 원장/관리자만 볼 수 있습니다.`

## 5. Owner 계정 확인

원장 계정에서 관리 탭을 확인했습니다.

- `시작 / 명단 / 수업 / 직원 / 문자 / 정책 / 리포트 / 이력` 메뉴가 표시되었습니다.
- 정책 카드에 `보조 발송 제한` 상태가 표시되었습니다.
- 관리 탭 모바일 390px에서 가로 overflow는 없었습니다.

## 6. 발견한 문제

### 문제 1. 발송 버튼은 assistant에게도 저장 후 노출됨

- 화면: 문자 바텀시트
- 계정: assistant
- 현상: 팔로업 저장 후 `문자 발송 테스트` 버튼이 표시됩니다.
- 실제 동작: 서버는 403으로 차단합니다.
- 영향도: 중간
- 우선순위: P2
- 판단: 보안 문제는 아니지만, 보조 선생님 입장에서는 눌러도 실패하는 버튼이 보이는 UX 혼란이 있을 수 있습니다.
- 추천 조치: `allow_assistant_send=false`인 assistant에게는 발송 버튼을 비활성화하고 `원장 권한이 필요합니다` 안내를 보여주는 후속 UX 개선을 검토합니다.
- 새 이슈 생성 필요 여부: 선택. 파일럿 중 혼란이 있으면 생성.

## 7. 수정 없이 보류해도 되는 항목

- assistant 발송 권한 모델 변경
- 실제 SMS 발송 테스트
- 보조 선생님 다중 배정 모델
- 발송 버튼 UX 개선

현재 서버 권한은 안전하게 동작하므로, 파일럿 전 필수 수정 대상은 아닙니다.

## 8. 즉시 이슈화 후보

이번 UAT 기준으로 P0/P1 신규 버그 이슈는 없습니다.

후속 UX 개선 후보는 아래와 같습니다.

- 이슈 제목: assistant 발송 버튼 권한 안내 개선 (T-625)
- 우선순위: P2
- 목표: 보조 선생님이 발송 제한 상태를 화면에서 바로 이해하게 함
- 구현 범위:
  - `allow_assistant_send=false`인 assistant에게 발송 버튼 비활성화
  - 버튼 아래 `보조 선생님은 원장 설정상 발송할 수 없습니다.` 안내 표시
  - 서버 403 차단은 유지
- 제외 범위:
  - 권한 모델 변경
  - 실제 SMS 발송
  - 설정 화면 변경
- 완료 기준:
  - assistant가 팔로업 저장 후 발송 제한 이유를 UI에서 확인 가능
  - owner/teacher 발송 흐름은 유지
- 테스트 기준:
  - `assistant@test.com` 모바일 390px 문자 바텀시트
  - `owner@test.com` 기존 발송 테스트 흐름
- 추천 브랜치명: `codex/t-625-assistant-send-button-copy`
- Codex 작업 지시문:
  - assistant 계정에서 `allow_assistant_send=false`일 때 문자 발송 테스트 버튼을 비활성화하고 권한 안내 문구를 추가해줘. 서버 권한 로직은 변경하지 마.

## 9. 최종 판단

- assistant 발송 제한은 파일럿 MVP 기준으로 안전하게 동작합니다.
- assistant는 담당 반 출석과 팔로업 draft 저장이 가능하고, 발송은 서버에서 403으로 차단됩니다.
- message log가 생성되지 않아 dry-run 발송도 발생하지 않았습니다.
- 6월 중순 기능 동결 기준에서 T-620은 완료 처리 가능합니다.
