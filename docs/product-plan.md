# 제품 기획서

## 1. 제품 정의

**Academy Follow-up**은 한국 학원 선생님과 원장이 학생별 수업 스케줄, 보강 가능 시간, 후속 안내를 한곳에서 확인하고 기록하는 학원 운영 SaaS입니다.

첫 버전은 거대한 학원관리시스템이 아닙니다. 핵심은 아래 한 가지입니다.

> 반별 출석 체크 -> 결석/지각 학생 확인 -> 학생 스케줄 확인 -> 학부모 문자 미리보기 -> 기록 저장

## 2. 파일럿 학원

- 학원명: 더배움프라임영수학원
- 업종: 학원 등 교습시설
- 주소: 충남 아산시 탕정면 한들물빛5로 17 트윈시티2 603호, 604호
- 위치: 탕정역 1번 출구에서 187m
- 영업시간: 23:00 운영 종료
- 네이버 플레이스 ID: 1298554109
- 방문자 리뷰: 16개
- 블로그 리뷰: 29개
- 예상 규모: 원생 약 200명, 선생님 약 16명

## 3. 문제

원생 200명 규모의 학원에서는 결석, 지각, 숙제 미완료, 재시험, 상담 권장 같은 후속 안내가 매일 반복됩니다.

현재 이 업무가 카톡, 문자, 엑셀, 선생님 개인 메모에 흩어지면 아래 문제가 생깁니다.

- 선생님마다 안내 기준과 말투가 달라짐
- 원장이 누가 어떤 학부모에게 연락했는지 알기 어려움
- 같은 학생에게 중복 발송하거나 꼭 보내야 할 안내가 누락됨
- 학부모 연락 기록이 학생별로 쌓이지 않음
- 보조 선생님이 체크한 내용이 정규 선생님/원장에게 늦게 전달됨
- 학부모 상담 전에 과거 연락 이력을 다시 찾기 어려움
- 학생별 수업 요일과 외부 일정이 흩어져 있어 보강 가능 시간을 바로 판단하기 어려움
- 결석 후 보강 안내를 보내려 해도 해당 학생의 다른 학원/논술/개인 일정을 다시 물어봐야 함
- 결석 학생을 선생님이 카톡방에 공유하고 데스크가 다시 확인해 연락해야 함
- 수업 시작 직후 미도착 학생이 지각인지 결석인지 애매해 즉시 문자 발송하기 어려움

## 4. 제품 가설

선생님이 수업 직후 학생별 스케줄을 보면서 10초 안에 팔로업 문자를 만들 수 있다면, 보강 안내 누락이 줄고 학원 내부 운영 기록이 정리됩니다.

원장에게 중요한 가치는 문자 발송 자체가 아니라 아래입니다.

- 오늘 어떤 학생에게 연락했는지 확인
- 반별/선생님별 팔로업 누락 확인
- 학생별 정규 수업/외부 일정/보강 가능 시간 확인
- 학생별 학부모 연락 히스토리 축적
- 반별 출석/지각/결석 현황 확인
- 선생님별 안내 문구와 기준 통일
- 상담 전 학생 상태 파악 시간 단축

## 5. 사용자

### 원장

- 전체 학원 현황을 봅니다.
- 발송 기록, 반별 현황, 선생님별 처리 여부를 확인합니다.
- 학생별 주간 스케줄과 보강 가능 시간을 확인합니다.
- 문자 템플릿과 발신 설정을 관리합니다.

### 정규 선생님

- 담당 반 학생의 결석, 재시험, 숙제 미완료 등을 학부모에게 안내합니다.
- 학생별 팔로업 기록을 확인합니다.
- 결석 학생의 스케줄을 보고 보강 안내 문자를 만듭니다.

### 보조 선생님

- 출석/숙제/재시험 대상 같은 수업 후 체크를 빠르게 남깁니다.
- 학원 설정에 따라 직접 발송하거나 초안만 작성합니다.

## 7. 핵심 플로우

1. 선생님 로그인
2. 내 학원 운영 보드 진입
3. 오늘 수업 반과 시간을 선택
4. 반별 출석부에서 학생 상태 체크
5. 결석/지각/확인 필요 학생 선택
6. 학생별 스케줄 확인
7. 필요한 경우 보강 날짜와 시간 선택
8. 문자 미리보기
9. 필요하면 문구 수정
10. 팔로업 기록 저장 또는 문자 발송
11. 보강 문자를 보낸 경우 스케줄 자동 등록 여부 확인
12. 원장/데스크 현황에 반영

표현 원칙은 학생을 낙인찍지 않는 것입니다. "위험 학생" 같은 표현은 쓰지 않고, "팔로업 필요", "확인 필요", "상담 권장"처럼 다음 행동 중심의 표현을 씁니다.

## 9. 문자 템플릿

결석:

```text
[{{학원명}}] 안녕하세요. 오늘 {{학생명}} 학생이 수업에 결석하여 안내드립니다.
확인 부탁드리며, 보강 일정이 필요한 경우 담당 선생님이 다시 안내드리겠습니다.
```

재시험:

```text
[{{학원명}}] 안녕하세요. {{학생명}} 학생은 오늘 수업 내용 중 재시험이 필요하여 안내드립니다.
다음 수업 전까지 오답을 다시 확인할 수 있도록 지도 부탁드립니다.
```

숙제 미완료:

```text
[{{학원명}}] 안녕하세요. {{학생명}} 학생이 이번 과제를 완료하지 못해 안내드립니다.
다음 수업 전까지 과제를 마무리할 수 있도록 확인 부탁드립니다.
```

상담 권장:

```text
[{{학원명}}] 안녕하세요. {{학생명}} 학생의 최근 학습 흐름에 대해 간단한 상담이 필요하여 안내드립니다.
가능하신 시간에 담당 선생님에게 연락 부탁드립니다.
```

보강 안내:

```text
[{{학원명}}] 안녕하세요. {{학생명}} 학생 보강 일정 안내드립니다.
확인 가능한 시간은 {{보강후보시간}}이며, 가능 여부를 회신 부탁드립니다.
```

## 9.1 학생 스케줄 MVP

원장 요청에 따라 학생별 스케줄 확인을 핵심 기능으로 올립니다. 단, 처음부터 완전한 캘린더를 만들지 않고 주간 반복 스케줄 중심으로 시작합니다.

### 포함

- 학생별 정규 수업 요일/시간
- 학생별 외부 일정 또는 보강 불가 시간
- 과목, 수업명, 담당 선생님
- 학생 상세에서 주간 시간표 보기
- 날짜 선택 후 해당 날짜의 수업/외부 일정 확인
- 선생님이 직접 보강 시작/종료 시간 입력
- 보강 문자 후 스케줄 자동 등록 확인

### 제외

- 실시간 드래그 앤 드롭 캘린더
- 강의실/좌석 배정
- 반복 예외 규칙 고도화
- 학부모 캘린더 공유
- 학원 간 학생 스케줄 공유

## 9.2 출석부 MVP

원장 피드백 기준으로 출석부는 파일럿 핵심 기능입니다. 현재 방식은 선생님이 반별로 학생 도착 여부를 확인한 뒤 결석을 카톡방에 공유하고, 데스크가 이를 보고 연락하는 구조입니다.

### 포함

- 날짜/반/수업 시간 기준 출석부
- 학생별 상태: 출석, 지각, 결석, 보강, 확인 필요
- 수업 시작 후 미도착 학생을 바로 결석 처리하지 않는 확인 필요 상태
- 결석/지각 상태에서 문자 초안 생성
- 출석 상태 수정 이력 또는 수정 가능 구조
- 원장/데스크가 오늘 결석/지각 연락 대상을 확인하는 현황

### 제외

- 자동 출입 태그/RFID/QR 체크인
- 학부모 실시간 출결 앱 알림
- 지문/얼굴 인식
- 결석 예측 자동화

## 10. 학원별 브랜딩

이 서비스는 여러 학원이 쓰는 SaaS가 되어야 하지만, 로그인 후 화면에서는 서비스 브랜드보다 학원 브랜드가 먼저 보여야 합니다.

### 원칙

- 상단에는 학원명과 로고를 보여줍니다.
- 주요 문구는 "대시보드"보다 "운영 보드", "오늘의 팔로업"처럼 내부 운영 언어를 씁니다.
- 문자 미리보기에는 SaaS 이름이 아니라 학원 발신명이 들어갑니다.
- 학원별 템플릿, 발신번호, 선생님명 표시 방식을 따로 관리합니다.

### 초기 커스터마이징

- 학원명
- 학원 slug
- 네이버 플레이스 ID
- 주소
- 발신명
- 발신번호
- 대표 색상
- 문자 템플릿

### 나중 확장

- 로고
- 커스텀 도메인
- 카카오 알림톡 채널
- 지점 관리
- 학부모용 리포트 브랜딩

## 11. 성공 기준

파일럿 2주 기준으로 아래를 봅니다.

- 선생님 2명 이상이 실제 수업 후 사용
- 하루 5건 이상 팔로업 기록 생성
- 발송 전 문구 수정 비율 확인
- 중복 발송/오발송 발생 여부 확인
- 원장이 발송 기록 화면을 실제로 확인하는지 확인
- 선생님이 기존 방식보다 빠르다고 느끼는지 확인
- 학부모 반응에서 부정적인 피드백이 없는지 확인

## 12. 확장 로드맵

전체 기능 범위는 [벤치마킹 기반 기능 맵](./benchmark-feature-map.md)을 기준으로 관리합니다.

### 1단계: 수업 후 팔로업 기록

- 학생 클릭
- 사유 선택
- 문자 미리보기
- 발송 기록
- 학생별 히스토리

### 2단계: 학원 기본 관리

- 반 등록/수정
- 학생 등록/수정
- 선생님/구성원 관리
- 학생별 주간 스케줄 등록

### 3단계: dry-run 발송과 안전장치

- dry-run 발송
- message log 저장
- 중복 발송 방지
- 학부모 번호 마스킹
- 실패 상태 표시

### 4단계: 스케줄 기반 운영 보드

- 학생별 주간 시간표: 완료
- 외부 일정/보강 불가 시간: 완료
- 날짜 선택 후 보강 가능성 확인: 완료
- 보강 문자 생성: 완료
- 보강 문자 후 스케줄 자동 등록: 완료

### 5단계: 운영 보드

- 오늘 발송 현황
- 반별/선생님별 필터
- 학생별 팔로업 히스토리
- 템플릿 관리
- CSV 학생 일괄 등록

### 6단계: 학습 케어 기록

- 출석 체크
- 결석/지각 문자 연결
- 확인 필요 상태와 지각 예외 처리
- 원장/데스크 출석 현황
- 숙제 체크
- 재시험 완료 여부
- 상담 기록
- 확인 필요 학생 리스트

### 7단계: 학부모 커뮤니케이션

- 주간 학부모 리포트
- 상담 준비 리포트
- 카카오 알림톡
- 학부모용 공유 링크

### 8단계: 상용화 운영

- 플랫폼 관리자 콘솔
- 가입/워크스페이스 온보딩
- 플랜/사용량 관리
- 결제/구독
- 고객 지원/문의
- 개인정보/보안 운영 문서

## 13. 가격 가설

초기 상용화 기준입니다.

- Starter: 월 5만~10만원, 학생 100명 이하
- Standard: 월 15만~25만원, 학생 300명 이하
- Pro: 월 30만원 이상, 다중 지점/고급 리포트

문자 발송 비용은 월 요금에 일부 포함하거나 별도 과금합니다.
