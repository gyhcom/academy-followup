# 제품 기획 초안

## 1. 문제

원생 200명, 선생님 16명 규모의 학원에서는 수업 후 학부모 팔로업이 반복적으로 발생합니다.

- 결석 안내
- 지각 안내
- 숙제 미완료 안내
- 재시험 안내
- 상담 권장
- 준비물 미지참 안내
- 수업 태도 관련 안내

현재 이 업무가 카톡, 문자, 엑셀, 선생님 개인 메모에 흩어지면 아래 문제가 생깁니다.

- 선생님마다 안내 기준과 말투가 달라짐
- 원장이 누가 어떤 학부모에게 연락했는지 알기 어려움
- 같은 학생에게 중복 발송하거나, 반대로 꼭 보내야 할 안내가 누락됨
- 학부모 연락 기록이 학생별로 쌓이지 않음
- 보조 선생님이 체크한 내용이 정규 선생님/원장에게 늦게 전달됨

## 2. 제품 가설

선생님이 학생 이름을 클릭하고 사유만 선택하면, 학부모에게 보낼 한국어 안내 문구가 자동으로 생성되고 발송 기록이 남는다면 학원 운영 시간이 줄고 팔로업 누락이 줄어든다.

## 3. 첫 번째 MVP

### 핵심 플로우

1. 선생님 로그인
2. 오늘 수업 반 선택
3. 학생 목록 확인
4. 학생 이름 클릭
5. 사유 선택
6. 문자 미리보기
7. 필요하면 문구 수정
8. 발송
9. 발송 기록 저장

### 기본 사유

- 결석
- 지각
- 숙제 미완료
- 재시험 필요
- 준비물 미지참
- 수업 태도 확인 필요
- 상담 권장

### 문자 예시

결석:

```text
[{{학원명}}] 안녕하세요. 오늘 {{학생명}} 학생이 수업에 결석하여 안내드립니다.
확인 부탁드리며, 보강 일정이 필요한 경우 담당 선생님이 다시 안내드리겠습니다.
```

재시험:

```text
[{{학원명}}] 안녕하세요. {{학생명}} 학생은 오늘 수업 내용 중 재시험이 필요하여 안내드립니다.
다음 수업 전까지 오답을 다시 확인할 수 있도록 지도 부탁드립니다.
```

숙제 미완료:

```text
[{{학원명}}] 안녕하세요. {{학생명}} 학생이 이번 과제를 완료하지 못해 안내드립니다.
다음 수업 전까지 과제를 마무리할 수 있도록 확인 부탁드립니다.
```

## 4. 원장 화면

원장은 발송 자체보다 운영 현황을 보고 싶어할 가능성이 큽니다.

- 오늘 발송 건수
- 사유별 발송 건수
- 선생님별 발송 건수
- 반별 발송 건수
- 최근 발송 내역
- 학생별 연락 히스토리
- 중복 발송 경고

## 5. 확장 방향

MVP 이후에는 문자 발송 도구에서 학원 운영 CRM으로 확장합니다.

### 2차

- 학생별 팔로업 히스토리
- 반별 필터
- 선생님별 담당 반 권한
- 템플릿 관리
- CSV 학생 일괄 등록
- 중복 발송 방지

### 3차

- 출석 체크
- 숙제 체크
- 재시험 완료 여부
- 상담 기록
- 주간 학부모 리포트
- 카카오 알림톡
- 학원별 요금제

## 6. 제품 원칙

- 한국어 우선
- 선생님 입력은 10초 안에 끝나야 함
- 발송 전에는 반드시 미리보기를 보여줌
- 학생을 낙인찍는 표현을 쓰지 않음
- "위험 학생" 대신 "팔로업 필요", "케어 필요", "상담 권장" 같은 표현 사용
- 처음부터 학원관리시스템 전체를 만들지 않음



## 7. 학원별 브랜딩 / 내 학원 느낌

이 서비스는 여러 학원이 쓰는 SaaS가 되어야 하지만, 로그인 후 화면에서는 서비스 브랜드보다 학원 브랜드가 먼저 보여야 합니다.

### 원칙

- 로그인 후 첫 화면 상단에는 학원명과 학원 로고를 보여줍니다.
- 주요 문구는 "대시보드"보다 "우리 학원 운영 보드", "오늘의 팔로업"처럼 내부 운영 언어를 씁니다.
- 문자 미리보기에는 SaaS 이름이 아니라 학원 발신명이 들어갑니다.
- 학원별 템플릿, 발신번호, 선생님명 표시 방식을 따로 관리합니다.

### 초기 커스터마이징

- 학원명
- 학원 slug
- 네이버 플레이스 ID
- 발신명
- 발신번호
- 대표 색상
- 문자 템플릿

### 나중 확장

- 로고
- 커스텀 도메인
- 카카오 알림톡 채널
- 지점 관리
- 학부모용 리포트 브랜딩
