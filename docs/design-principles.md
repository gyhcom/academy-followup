# 디자인 원칙

Academy Follow-up의 디자인 기준은 "예쁜 SaaS"가 아니라 학원 선생님이 수업 직후 빠르게 처리하는 운영 도구입니다.

## 1. 기준 자료

다음 기준을 제품 판단의 근거로 사용합니다.

- WCAG 2.2: 접근성 최소 기준. 초기 목표는 AA 수준입니다.
- WAI-ARIA Authoring Practices Guide: 탭, 목록, 다이얼로그, 메뉴 같은 인터랙션 패턴의 키보드/스크린리더 동작 기준입니다.
- GOV.UK Design System Accessibility: 업무형 서비스에서 명확성, 오류 방지, 키보드 접근성을 판단할 때 참고합니다.
- Apple Human Interface Guidelines: 정보 계층, 일관성, 인터페이스 문구 기준을 참고합니다.
- Material Design Accessibility: 색 대비, 터치 영역, 색상 외 상태 표현 기준을 참고합니다.

## 2. 제품 디자인 방향

이 앱은 마케팅 사이트나 화려한 CRM이 아닙니다.

디자인 목표:

- 선생님이 10초 안에 팔로업 문자를 만들 수 있어야 합니다.
- 원장이 오늘 처리 상태를 한눈에 볼 수 있어야 합니다.
- 학생이나 학부모에게 불편한 표현을 쓰지 않아야 합니다.
- 실수 발송을 막는 확인 장치가 먼저 보여야 합니다.
- 여러 학원이 써도 각 학원 안에서는 "내 학원 시스템"처럼 보여야 합니다.

## 3. 핵심 화면 구조

로그인 후 첫 화면은 아래 흐름을 기본으로 합니다.

```text
오늘 운영 보드
  -> 반 선택
  -> 학생 선택
  -> 사유 선택
  -> 문자 미리보기
  -> 발송 또는 dry-run 저장
```

원칙:

- 핵심 업무는 한 화면 안에서 끝냅니다.
- 페이지 이동보다 선택 상태와 작업 패널을 우선합니다.
- 오늘 처리할 일, 선택한 학생, 다음 버튼이 항상 보여야 합니다.
- 원장/선생님 권한에 따라 보이는 반과 액션을 다르게 합니다.

## 4. 접근성 기준

초기 구현에서 반드시 지킬 기준입니다.

- 본문 텍스트 대비는 4.5:1 이상을 목표로 합니다.
- 아이콘, 버튼 경계, 상태 표시 대비는 3:1 이상을 목표로 합니다.
- 주요 클릭 영역은 최소 44px, 가능하면 48px 이상으로 둡니다.
- 버튼, 입력, textarea에는 명확한 label 또는 aria-label을 둡니다.
- placeholder만으로 입력 목적을 설명하지 않습니다.
- 키보드 Tab 이동 순서가 실제 업무 순서와 같아야 합니다.
- focus-visible 상태가 눈에 띄어야 합니다.
- 오류 메시지는 "무엇이 문제인지"와 "다음에 무엇을 해야 하는지"를 함께 말합니다.
- 색상만으로 상태를 구분하지 않고 텍스트나 아이콘을 같이 씁니다.

## 5. 문구 기준

학원 내부 사용자가 바로 이해하는 말만 씁니다.

좋은 표현:

- 오늘의 팔로업
- 결석 안내 만들기
- 문자 미리보기
- 발송 기록
- 확인 필요

피할 표현:

- 위험 학생
- 엔티티 생성
- 워크플로우 프로세스
- 메시지 오브젝트
- CRM 액션

## 6. 화면 밀도와 시각 스타일

운영 도구이므로 정보 밀도는 적당히 높게 유지합니다.

- 큰 히어로 영역을 만들지 않습니다.
- 같은 정보가 반복되는 카드 그리드를 남발하지 않습니다.
- 반/학생/작업 패널처럼 업무 단위가 분명한 레이아웃을 사용합니다.
- 흰색 표면, 얇은 경계선, 절제된 그림자를 기본으로 합니다.
- 초록은 완료/정상, 주황은 확인 필요, 빨강은 실패/주의, 회색은 대기 상태에 사용합니다.

## 7. 모바일 기준

모바일에서는 같은 정보를 억지로 3열에 넣지 않습니다.

- 반 목록 -> 학생 목록 -> 작업 패널 순서로 세로 배치합니다.
- 버튼은 엄지로 누르기 쉬운 높이를 유지합니다.
- 문자 본문은 작은 화면에서도 최소 5줄 이상 보이게 합니다.
- 헤더와 로그아웃 버튼은 줄바꿈되어도 겹치지 않아야 합니다.

## 8. 검증 기준

디자인 변경 후 아래를 확인합니다.

- `/login`에서 로그인 가능
- `/app`에서 학원명과 권한 표시
- 반 선택 가능
- 학생 선택 가능
- 사유 선택 가능
- 문자 초안이 학생/사유에 따라 바뀜
- 키보드 Tab 이동 가능
- 모바일 폭에서 가로 스크롤 없음
