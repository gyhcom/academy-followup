# 가격 및 무료 정책 전략

이 문서는 Academy Follow-up의 초기 무료/유료 기능 경계와 과금 가설을 정리합니다.

핵심 원칙은 다음과 같습니다.

> 학원은 문자 발송 자체 때문에 돈을 내는 것이 아니라, 출석·보강·히스토리·원장 운영 확인 같은 운영 관리 가치 때문에 돈을 낸다.

따라서 문자 기능은 초기 유입과 파일럿 검증을 위한 입구 역할로 사용하고, 실제 과금은 운영 관리 기능 중심으로 설계합니다.

## 1. 제품 포지션

Academy Follow-up은 거대한 학원관리 ERP를 바로 대체하는 제품이 아닙니다.

초기 포지션:

```text
수업 후 발생하는 결석, 지각, 보강, 연락 업무를 한 화면에서 처리하는 운영 보드
```

핵심 가치:

- 결석/지각 연락 누락 감소
- 보강 일정 관리
- 학생별 히스토리 축적
- 원장 운영 확인
- 선생님 간 처리 기준 통일

## 2. 무료 전략

초기 파일럿과 도입 장벽 감소를 위해 문자 기능 일부를 무료로 제공합니다.

### 무료 정책 목적

- 학원이 쉽게 테스트하게 만들기
- 선생님이 실제로 사용하게 만들기
- 기존 카톡/엑셀 흐름을 대체하게 만들기
- 운영 데이터와 사용 패턴을 확보하기

### 무료 정책 원칙

- 문자 자체가 제품의 핵심이 되지 않게 합니다.
- 운영 흐름 안에서 문자 기능이 자연스럽게 연결되게 합니다.
- 무제한 무료 정책은 피합니다.
- 실제 문자 비용과 남용 가능성을 고려합니다.

## 3. 추천 무료 범위

### 무료

- 로그인
- 기본 출석 체크
- 학생 상태 변경
- 문자 초안 생성
- dry-run 발송
- 기본 학생 히스토리
- 학생 50명 이하
- 선생님 2명 이하

### 제한 무료 또는 체험

- 실제 SMS/LMS 월 100~300건
- CSV 업로드 제한
- 최근 기록만 조회 가능

## 4. 추천 유료 기능

### 운영 관리 기능

- 원장 미처리 보드
- 선생님별 처리 현황
- 학생별 전체 히스토리
- 보강 스케줄 자동 등록
- 보강 중복 방지
- CSV 일괄 등록
- 권한 관리
- 감사 로그
- 학원별 설정

### 상용화 이후 확장 기능

- 다중 지점
- 카카오 알림톡
- 플랫폼 관리자 기능
- 고급 통계
- 학부모 리포트
- 커스텀 도메인

## 5. 추천 플랜 구조

### Starter

추천 가격:

- 월 5만 ~ 10만원

추천 대상:

- 학생 100명 이하 학원

포함:

- 기본 출석부
- 문자 초안
- dry-run
- 제한된 실제 문자
- 기본 히스토리

### Standard

추천 가격:

- 월 15만 ~ 25만원

추천 대상:

- 학생 100~300명 학원

포함:

- 원장 운영 보드
- 보강 스케줄 관리
- CSV 등록
- 선생님 권한 관리
- 전체 히스토리

### Pro

추천 가격:

- 월 30만원 이상

추천 대상:

- 다중 반/지점 운영 학원

포함:

- 다중 지점
- 고급 리포트
- 고급 운영 통계
- 우선 지원

## 6. 실제 문자 비용 전략

초기에는 아래 전략을 권장합니다.

### 1단계

- dry-run 중심 운영
- 실제 문자 비용 최소화
- 테스트 번호 기반 검증

### 2단계

- 제한된 실제 문자 허용
- 월 기본 제공 건수 포함
- 초과 건수 별도 계산

### 3단계

- 문자 발송량 기반 과금 옵션 검토
- 카카오 알림톡 검토

## 7. 무료 사용자 남용 방지

무료 문자 기능만 사용하는 사용자를 막기 위해 아래 제한을 둡니다.

- 학생 수 제한
- 월 발송 건수 제한
- 선생님 계정 제한
- 히스토리 기간 제한
- 고급 운영 화면 제한

## 8. 유료 전환 판단 기준

아래 행동이 반복되면 유료 전환 가능성이 높습니다.

- 원장이 매일 오늘 현황을 확인함
- 선생님이 수업 후 카톡보다 시스템을 먼저 사용함
- 보강 일정을 시스템 기준으로 관리함
- 학생별 과거 기록을 상담 전에 확인함
- 결석/지각 연락 누락이 줄었다고 느낌

## 9. 파일럿 예외 정책

친구 학원 파일럿은 아래 정책을 적용합니다.

- 실제 비용 청구 없음
- 기능 제한 최소화
- 실제 운영 피드백 우선
- 기능 추가보다 운영 흐름 검증 우선

단, 파일럿에서도 아래는 반드시 검증합니다.

- 중복 발송 방지
- 수신자 확인
- 권한 분리
- 모바일 사용성
- 학생 히스토리 저장

## 10. 장기 방향

장기적으로 Academy Follow-up은 아래 방향으로 확장 가능합니다.

```text
문자 도구
-> 운영 보드
-> 학원 운영 CRM
-> 학습 케어 플랫폼
```

하지만 초기 단계에서는 절대 범위를 과도하게 넓히지 않습니다.

현재 최우선 목표:

> 친구 학원에서 실제 수업 후 흐름이 반복 사용되는가
