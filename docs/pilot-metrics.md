# 파일럿 검증 지표

이 문서는 Academy Follow-up 파일럿에서 무엇을 성공으로 볼지 정의합니다.

기능 개수가 아니라 실제 학원 운영에서 아래 문제가 줄어드는지를 기준으로 판단합니다.

- 결석/지각 연락 누락
- 보강 일정 누락
- 선생님 간 처리 기준 차이
- 원장의 운영 확인 어려움
- 카톡/엑셀/개인 메모 분산

## 1. 핵심 가설

### 가설 1

출석 체크와 문자 초안을 한 화면에서 처리하면 선생님의 후속 연락 속도가 빨라진다.

### 가설 2

학생 스케줄과 보강 후보 시간을 바로 볼 수 있으면 보강 일정 누락이 줄어든다.

### 가설 3

원장용 오늘 현황 화면이 있으면 카톡방 기반 운영 확인이 줄어든다.

### 가설 4

dry-run 기반 검증만으로도 실제 운영 흐름 문제를 충분히 발견할 수 있다.

## 2. 2주 파일럿 성공 기준

### 최소 성공 기준

- 선생님 2명 이상 실제 사용
- 주 3회 이상 사용
- 하루 평균 팔로업 기록 5건 이상
- 원장이 오늘 현황 화면을 주 3회 이상 확인
- 중복 발송 0건
- 오발송 0건
- 실제 업무 흐름에서 사용 중단 없이 2주 유지

### 긍정 신호

- 기존 카톡 공유보다 빠르다는 피드백
- 보강 일정 등록이 쉬워졌다는 피드백
- 원장이 선생님별 처리 상태를 보기 시작함
- 학생 히스토리를 실제 상담 전에 확인함

### 위험 신호

- 문자 초안만 사용하고 기록 저장은 거의 안 함
- 보강 기능 사용률 낮음
- 원장 화면 조회 거의 없음
- 선생님이 수업 후 사용을 귀찮아함
- 모바일에서 입력 흐름이 느림

## 3. 측정 이벤트

초기에는 PostHog 또는 자체 이벤트 로그 테이블을 사용합니다.

추천 이벤트:

```text
login_success
attendance_status_changed
message_preview_generated
followup_created
message_send_requested
message_send_dry_run
makeup_schedule_created
dashboard_viewed
student_history_viewed
```

## 4. 이벤트 로그 구조 예시

```text
usage_events
- id
- academy_id
- user_id
- event_name
- entity_type
- entity_id
- metadata jsonb
- created_at
```

metadata 예시:

```json
{
  "studentId": "student_123",
  "classId": "class_1",
  "attendanceStatus": "absent",
  "recipientType": "parent"
}
```

## 5. 운영 지표

### 사용 지표

| 지표 | 목표 |
|---|---|
| 하루 팔로업 기록 수 | 5건 이상 |
| 선생님별 주간 사용 횟수 | 주 3회 이상 |
| 문자 초안 생성 수 | 꾸준히 증가 |
| 원장 현황 화면 조회 수 | 주 3회 이상 |
| 보강 일정 등록 수 | 실제 결석 발생 대비 증가 |

### 안정성 지표

| 지표 | 목표 |
|---|---|
| 중복 발송 | 0 |
| 오발송 | 0 |
| 잘못된 수신자 선택 | 0 |
| 보강 중복 등록 | 0 |
| 비로그인 API 접근 | 전부 401 |

### UX 지표

| 지표 | 목표 |
|---|---|
| 출석 상태 변경 시간 | 10초 이내 |
| 문자 초안 생성 시간 | 5초 이내 |
| 보강 등록 흐름 | 1분 이내 |
| 모바일 가로 스크롤 | 없음 |

## 6. 원장 인터뷰 질문

- 오늘 누락 학생을 더 쉽게 찾을 수 있었나요?
- 선생님별 처리 상태를 보는 것이 도움이 되었나요?
- 실제로 돈을 낸다면 어떤 기능 때문에 낼 것 같나요?
- 문자 기능보다 중요한 기능은 무엇인가요?
- 현재 방식(카톡/엑셀)으로 다시 돌아가고 싶은가요?

## 7. 선생님 인터뷰 질문

- 수업 직후 사용하기에 빠른가요?
- 어떤 단계에서 가장 많이 멈추나요?
- 문자 초안 표현이 자연스러운가요?
- 보강 일정 찾는 흐름이 편한가요?
- 모바일에서 불편한 영역은 어디인가요?

## 8. 무료/유료 판단 기준

문자 발송 자체보다 아래 기능에서 유료 전환 가능성을 확인합니다.

- 원장 미처리 확인 보드
- 학생별 히스토리
- 보강 스케줄 관리
- 선생님별 처리 현황
- CSV 일괄 등록
- 권한 관리

파일럿에서 아래가 반복되면 유료 가능성이 있습니다.

- 원장이 매일 현황 화면을 확인
- 선생님이 카톡 대신 시스템을 먼저 열어봄
- 보강 일정이 실제로 시스템 기준으로 관리됨
- 학생별 과거 기록을 상담 전에 다시 확인함

## 9. 파일럿 종료 후 액션

### 계속 개발

- 실제 사용 반복 확인
- 원장 가치 확인
- 운영 누락 감소 확인

### 기능 축소

- 거의 사용되지 않는 기능 제거
- 클릭 수 감소
- 모바일 흐름 단순화

### 상용화 준비

- 실제 문자 발송 제한 오픈
- 가격 정책 확정
- 두 번째 학원 파일럿 시작
- 개인정보/운영 문서 정리
